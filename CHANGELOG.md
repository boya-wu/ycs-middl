# Changelog

格式基於 [Keep a Changelog](https://keepachangelog.com/zh-TW/)，版本號遵循 [Semantic Versioning](https://semver.org/lang/zh-TW/)。

## [Unreleased]

### Added

- 新增「版本更新內容」頁面（/changelog），可直接查看各版本更新摘要
- Dashboard 側欄新增「更新內容」入口，並顯示目前版本號
- 建立發版流程與工具，讓每次更新都有對應的版本紀錄可查閱

### Changed

- 登入頁工號欄位文案更新，提示更清楚
- 強化 AI 發版判斷流程：AI 現在會依據改動性質（bug 修正 / 新增功能 / 破壞性變更）自動建議應升 patch、minor 或 major，並說明理由

## [0.5.0-beta.1] - 2026-04-14

### Added

- SR 任務頁面新增標題顯示與已認領工時明細
- 改用員工工號登入，登入後身份持續保持
- 移除預設「匯入工時小於 5 分鐘」的過濾機制
- 新增人員名冊匯入功能，支援批次建立員工資料

## [0.4.0-beta.1] - 2026-04-13

### Added

- 新增專案（PY）與任務（SR）的基礎維護頁面
- 新增 Dashboard PIP 自我檢核流程
- 全面導入統一樣式系統，介面呈現更一致
- 下拉選單元件升級，操作更流暢
- 建立新認領記錄時，「可請款」選項預設為開啟

### Changed

- 全站術語統一：「裁決」改為「認領」，用詞更直覺
- 改善專案任務資料的即時同步，減少手動刷新

## [0.3.0-beta.1] - 2026-03-24

### Changed

- 改善認領頁面的資料處理效率
- 改善環境設定缺失時的錯誤提示，避免系統靜默失敗
- 更新系統各頁面的標題與說明文字
- 建立資料庫結構版本管理機制（內部工具），確保跨環境一致性

## [0.2.0-beta.1] - 2026-02-11

### Added

- 認領彙整頁面新增認領原因欄位，並解決重複記錄問題
- 切換分頁回來後自動更新資料，無需手動重整
- 新增「已認領」分頁，可快速瀏覽已認領工時的彙整摘要

### Fixed

- 修復認領記錄與工時匯入的資料同步異常

## [0.1.0-beta.1] - 2026-01-25

### Added

- 上線初版工時上傳頁面
- 完成基礎建設，串接後端資料庫

### Fixed

- 修正認領操作後頁面未即時更新的問題
